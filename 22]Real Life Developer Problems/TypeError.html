TypeError

This example demonstrates how to handle TypeError in JavaScript.

Incorrect Approach

Accessing a property of an undefined object will result in a TypeError.

            var obj = {};
            obj.name; // undefined
            obj.name.age; // TypeError: cannot read property 'age' of undefined

Correct Approach

Using optional chaining (?.) to safely access nested properties.

            var obj = {};
            obj?.name;
            obj?.name?.age;

Comments

  • The incorrect approach directly accesses the property of an undefined object, leading to a TypeError.
  • The correct approach uses optional chaining to safely access nested properties, avoiding the TypeError.
  • Optional chaining (?.) is a feature in JavaScript that allows you to safely access deeply nested properties without having to explicitly check if each reference in the chain is valid.
  • This can help prevent runtime errors and make your code more robust and easier to read.
