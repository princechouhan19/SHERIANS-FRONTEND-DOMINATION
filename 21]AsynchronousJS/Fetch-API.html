Fetch API

kyuki ye internet par jaayega aur fir data ko lekar aayega to ismein time lagta hai to by default hi js mein fetch ko async banaaya gaya hai kyuki fetch ka kaam hai data laana wo bhi kisi url se ab aisa ho skta hai us url ki website slow ho, to data laane mein time lage aur agar fetch synchronous hota to uske baad ka code tab tak nahi chalta jab tak uska data nahi aajata, wh is a big problem, poora code atak sakta tha isliye fetch ko async banaaya gaya hai, isse hi fetch data laane mein time lagta hai lekin baad mein hi data aayega to uske baad ka code chalta jayega, isse hi problem solve ho jata hai.

The Fetch API provides a modern, promise-based way to make network requests from the browser. It is asynchronous by design, ensuring that your code remains responsive while data is being retrieved over the internet. 


// Basic fetch syntax
fetch(url)
  .then(response => response.json()) // or .text(), .blob(), etc.
  .then(data => { /* use data */ })
  .catch(error => { /* handle error */ });
